15
16Misc ramblings:
17
18The terms axis and joint are used inconsistently throughout EMC.
19For all new code, the usages are as follows:
20
21 axis - one of the nine degrees of freedom, x, y, z, a, b, c, u, v, w
22 these refer to axes in Cartesian space, which may or
23 may not match up with joints (see below). On Cartesian
24 machines they do match up, but for hexapods, robots, and
25 other non-Cartesian machines they don't.
26 joint - one of the physical degrees of freedom of the machine
27 these might be linear (leadscrews) or rotary (rotary
28 tables, robot arm joints). There can be any number of
29 joints. The kinematics code is responsible for translating
30 from axis space to joint space and back.
31
32There are three main kinds of data needed by the motion controller
33
341) data shared with higher level stuff - commands, status, etc.
352) data that is local to the motion controller
363) data shared with lower level stuff - hal pins
37
38In addition, some internal data (2) should be shared for trouble
39shooting purposes, even though it is "internal" to the motion
40controller. Depending on the type of data, it can either be
41treated as type (1), and made available to the higher level
42code, or it can be treated as type (3), and made available to
43the hal, so that halscope can monitor it.
44
45This file should ONLY contain structures and declarations for
46type (1) items - those that are shared with higher level code.
47
48Type (2) items should be declared in mot_priv.h, along
49with type (3) items.
50
51In the interest of retaining my sanity, I'm not gonna attempt
52to move everything to its proper location yet....
53
54However, all new items will be defined in the proper place,

57

282/*
283 motion status flag structure-- looks like:
284
285 MSB LSB
286 v---------------v------------------v
287 | | | | T | CE | C | IP | EN |
288 ^---------------^------------------^
289
290 where:
291
292 EN is 1 if calculations are enabled, 0 if not
293 IP is 1 if all joints in position, 0 if not
294 C is 1 if coordinated mode, 0 if in free mode
295 CE is 1 if coordinated mode error, 0 if not
296 T is 1 if we are in teleop mode.
297 */

306/* joint flag type */

308/*
309 joint status flag structure-- looks like:
310
311 MSB LSB
312 ----------v-----------------v--------------------v-------------------v
313 | AF | FE | AH | HD | H | HS | NHL | PHL | - | - | ER | IP | AC | EN |
314 ----------^-----------------^--------------------^-------------------^
315
316
317 x = unused
318
319 where:
320
321 EN is 1 if joint amplifier is enabled, 0 if not
322 AC is 1 if joint is active for calculations, 0 if not
323 IP is 1 if joint is in position, 0 if not (free mode only)
324 ER is 1 if joint has an error, 0 if not
325
326 PHL is 1 if joint is on maximum hardware limit, 0 if not
327 NHL is 1 if joint is on minimum hardware limit, 0 if not
328
329 HS is 1 if joint home switch is tripped, 0 if not
330 H is 1 if joint is homing, 0 if not
331 HD is 1 if joint has been homed, 0 if not
332 AH is 1 if joint is at home position, 0 if not
333
334 FE is 1 if joint exceeded following error, 0 if not
335 AF is 1 if amplifier is faulted, 0 if not
336
337Suggestion: Split this in to an Error and a Status flag register..
338 Then a simple test on each of the two flags can be performed
339 rather than testing each bit... Saving on a global per joint
340 fault and ready status flag.
341 */
