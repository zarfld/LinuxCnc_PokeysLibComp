50/*
51 general-purpose timer, which can be used for waiting until a
52 synchronous time tick, slept on for any period at all, or to
53 obtain a time in system clock ticks from creation of the timer.
54 */
55
56/*
57Programmers can create RCS_TIMER objects to synchronize to the system clock or to some other event(s).
58
59To synchronize a cyclic process to the system clock:
60
61Initialize the RCS_TIMER object with the cycle period.
62Call RCS_TIMER::wait() at the end of each cycle.
63
64The cycle period will be rounded up to the resolution of the system clock or
65the most precise time measuring or sleeping function available for the given
66platform. RCS_TIMER::wait() will wait the remainder of the cycle period since
67the last call. The units for the cycle time are seconds.
68*/
69/*
70
71To synchronize to some other event(s):
72
73 Create a function that takes a (void *) as an argument and returns an int.
74 Initialize the RCS_TIMER object with a cycle period used only for diagnostics,
75 the address of the function and a parameter for the function.
76 Use RCS_TIMER::wait().
77
78The user's function should return 0 when the event to synchronize to occurs or -1
79if an error occurs. The argument passed to the users function will be whatever was
80passed as the third parameter to the constructor of the RCS_TIMER or NULL if no
81third argument is given. This argument could be used by the synchronizing function
82to know which timer is calling it if the synchronization function is called by more
83than one timer. Nothing will force the function to return within the cycle period but
84there are ways to check if the function took longer than the cycle period after it returns.
85*/
