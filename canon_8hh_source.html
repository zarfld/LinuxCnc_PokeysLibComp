25/*
26 This is the header file that all applications that use the
27 canonical commands for three- to nine-axis machining should include.
28
29 Three mutually orthogonal (in a right-handed system) X, Y, and Z axes
30 are always present. In addition, there may be zero to three rotational
31 axes: A (parallel to the X-axis), B (parallel to the Y-axis), and C
32 (parallel to the Z-axis). Additionally there may be zero to three linear
33 axes: U, V and W.
34
35 In the functions that use rotational axes, the axis value is that of a
36 wrapped linear axis, in degrees.
37
38 It is assumed in these activities that the spindle tip is always at
39 some location called the 'current location,' and the controller always
40 knows where that is. It is also assumed that there is always a
41 'selected plane' which must be the XY-plane, the YZ-plane, or the
42 ZX-plane of the machine.
43*/

271/*
272
273SET_FEED_RATE sets the feed rate that will be used when the spindle is
274told to move at the currently set feed rate. The rate is either:
2751. the rate of motion of the tool tip in the workpiece coordinate system,
276 which is used when the feed_reference mode is "CANON_WORKPIECE", or
2772. the rate of motion of the tool tip in the XYZ axis system, ignoring
278 motion of other axes, which is used when the feed_reference mode is
279 "CANON_XYZ".
280
281The units of the rate are:
282
2831. If the feed_reference mode is CANON_WORKPIECE:
284length units (inches or millimeters according to the setting of
285CANON_UNITS) per minute along the programmed path as seen by the
286workpiece.
287
2882. If the feed_reference mode is CANON_XYZ:
289A. For motion including one rotational axis only: degrees per minute.
290B. For motion of two or three rotational axes with X, Y, Z, U, V, and W
291 axes not moving, the rate is applied as follows. Let dA, dB, and dC
292 be the angles in degrees through which the A, B, and C axes,
293 respectively, must move. Let D = sqrt(dA*dA + dB*dB + dC*dC).
294 Conceptually, D is a measure of total angular motion, using the usual
295 Euclidean metric. Let T be the amount of time required to move through
296 D degrees at the current feed rate in degrees per minute. The
297 rotational axes should be moved in coordinated linear motion so that
298 the elapsed time from the start to the end of the motion is T plus any
299 time required for acceleration or deceleration.
300C. For motion of secondary linear axes (U, V, and/or W) with X, Y, and Z
301 axes not moving (with or without simultaneous rotational axis motion):
302 length units (inches or millimeters according to the setting of
303 CANON_UNITS) per minute in the UVW cartesian system.
304D. For motion involving one or more of the XYZ axes (with or without
305 simultaneous motion of other axes): length units (inches or
306 millimeters according to the setting of CANON_UNITS) per minute
307 along the programmed XYZ path.
308
309*/
310
311extern void SET_FEED_REFERENCE(CANON_FEED_REFERENCE reference);
312
313/*
314
315This sets the feed_reference mode to either CANON_WORKPIECE or
316CANON_XYZ.
317
318The CANON_WORKPIECE mode is more natural and general, since the rate
319at which the tool passes through the material must be controlled for
320safe and effective machining. For machines with more than the three
321standard XYZ axes, however, computing the feed rate may be
322time-consuming because the trajectories that result from motion in
323four or more axes may be complex. Computation of path lengths when
324only XYZ motion is considered is quite simple for the two standard
325motion types (straight lines and helical arcs).
326
327Some programming languages (rs274kt, in particular) use CANON_XYZ
328mode. In these languages, the task of dealing with the rate at which
329the tool tip passes through material is pushed back on the NC-program
330generator, where the computation of path lengths is (almost always in
3311995) an off-line activity where speed of calculation is not critical.
332
333In CANON_WORKPIECE mode, some motions cannot be carried out as fast as
334the programmed feed rate would require because axis motions tend to
335cancel each other. For example, an arc in the YZ-plane can exactly
336cancel a rotation around the A-axis, so that the location of the tool
337tip with respect to the workpiece does not change at all during the
338motion; in this case, the motion should take no time, which is
339impossible at any finite rate of axis motion. In such cases, the axes
340should be moved as fast as possible consistent with accurate
341machining.
342
343It would be possible to omit the SET_FEED_REFERENCE command from the
344canonical commands and operate always in one mode or the other,
345letting the interpreter issue SET_FEED_RATE commands, if necessary to
346compensate if the NC language being interpreted used the other mode.
347
348This would create two disadvantages when the feed_reference mode
349assumed by the canonical commands differed from that assumed by the NC
350language being interpreted:
351
3521. The output code could have a lot of SET_FEED_RATE commands not
353found in the input code; this is a relatively minor consideration.
354
3552. If the interpreter reads a program in language which uses the
356CANON_XYZ mode and writes canonical commands in the CANON_WORKPIECE
357mode, both the interpreter and the executor of the output canonical
358commands would have to perform a lot of complex calculations. With the
359SET_FEED_REFERENCE command available, both do only simple calculations
360for the same motions.
361
362*/
363

406extern void ARC_FEED(int lineno,
407 double first_end, double second_end,
408 double first_axis, double second_axis, int rotation,
409 double axis_end_point,
410 double a, double b, double c,
411 double u, double v, double w);
412
413/* Move in a helical arc from the current location at the existing feed
414rate. The axis of the helix is parallel to the x, y, or z axis,
415according to which one is perpendicular to the selected plane. The
416helical arc may degenerate to a circular arc if there is no motion
417parallel to the axis of the helix.
418
4191. If the selected plane is the xy-plane:
420A. first_end is the x-coordinate of the end of the arc.
421B. second_end is the y-coordinate of the end of the arc.
422C. first_axis is the x-coordinate of the axis (center) of the arc.
423D. second_axis is the y-coordinate of the axis.
424E. axis_end_point is the z-coordinate of the end of the arc.
425
4262. If the selected plane is the yz-plane:
427A. first_end is the y-coordinate of the end of the arc.
428B. second_end is the z-coordinate of the end of the arc.
429C. first_axis is the y-coordinate of the axis (center) of the arc.
430D. second_axis is the z-coordinate of the axis.
431E. axis_end_point is the x-coordinate of the end of the arc.
432
4333. If the selected plane is the zx-plane:
434A. first_end is the z-coordinate of the end of the arc.
435B. second_end is the x-coordinate of the end of the arc.
436C. first_axis is the z-coordinate of the axis (center) of the arc.
437D. second_axis is the x-coordinate of the axis.
438E. axis_end_point is the y-coordinate of the end of the arc.
439
440If rotation is positive, the arc is traversed counterclockwise as
441viewed from the positive end of the coordinate axis perpendicular to
442the currently selected plane. If rotation is negative, the arc is
443traversed clockwise. If rotation is 0, first_end and second_end must
444be the same as the corresponding coordinates of the current point and
445no arc is made (but there may be translation parallel to the axis
446perpendicular to the selected plane and motion along the rotational axes).
447If rotation is 1, more than 0 but not more than 360 degrees of arc
448should be made. In general, if rotation is n, the amount of rotation
449in the arc should be more than ([n-1] x 360) but not more than (n x
450360).
451
452The radius of the helix is determined by the distance from the current
453location to the axis of helix or by the distance from the end location
454to the axis of the helix. It is recommended that the executing system
455verify that the two radii are the same (within some tolerance) at the
456beginning of executing this function.
457
458While the XYZ motion is going on, move the rotational axes so that
459they have always covered the same proportion of their total motion as
460a point moving along the arc has of its total motion.
461
462*/
463

466 double a, double b, double c,
467 double u, double v, double w);

564/* It is assumed that each cutting tool in the machine is assigned to a
565slot (intended to correspond to a slot number in a tool carousel).
566This command results in the tool currently in the spindle (if any)
567being returned to its slot, and the tool from the slot designated by
568slot_number (if any) being inserted in the spindle.
569
570If there is no tool in the slot designated by the slot argument, there
571will be no tool in the spindle after this command is executed and no
572error condition will result in the controller. Similarly, if there is
573no tool in the spindle when this command is given, no tool will be
574returned to the carousel and no error condition will result in the
575controller, whether or not a tool was previously selected in the
576program.
577
578It is expected that when the machine tool controller is initialized,
579the designated slot for a tool already in the spindle will be
580established. This may be done in any manner deemed fit, including
581(for, example) recording that information in a persistent, crash-proof
582location so it is always available from the last time the machine was
583run, or having the operator enter it. It is expected that the machine
584tool controller will remember that information as long as it is
585not re-initialized; in particular, it will be remembered between
586programs.
587
588For the purposes of this command, the tool includes the tool holder.
589
590For machines which can carry out a select_tool command separately from
591a change_tool command, the select_tool command must have been given
592before the change_tool command, and the value of slot must be the slot
593number of the selected tool. */
