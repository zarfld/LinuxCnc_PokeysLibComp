Detailed Description

Base routines for setting up HAL pins and parameters for PoKeys Pulse Engine v2.

This file contains the foundational helper functions used during HAL component initialization. It provides functions to create HAL pins and parameters that reflect the internal state and control options of the PoKeys Pulse Engine v2 (PEv2), including:

  • Input and output signals for probing, limit switches, home switches, and emergency stop.
  • Motion parameters such as speed, acceleration, and homing behavior.
  • Soft limits and axis enable controls.

These routines are shared by both userspace and realtime components and are called from the respective component setup logic to populate the HAL interface with the required signal endpoints.

int32_t PEv2_HomingStateSyncedTrigger ( sPoKeysDevice *  dev,

pokeys_home_status_t  RequiredState,

Trigger state transitions for all axes in a homing sequence synchronously.

This function checks all axes in a given homing sequence and evaluates whether all of them are in the required state. If so, it transitions them to the next state and performs related hardware actions (e.g., arming encoders, setting positions, starting movement).

The function logs state mismatches and suppresses repeated messages using memory buffers.

Synchronization ensures that axes in the same sequence only transition when all others are also ready, making it suitable for multi-axis machines requiring coordinated homing.

Parameters
devPointer to the PoKeys device structure.
seqHoming sequence number to process.
RequiredStateExpected current homing status for all axes in sequence.
NextStateNext state to transition to if all axes match RequiredState.
Return values
0Success, state transition triggered.
1Transition not triggered (not all axes ready, invalid state, etc).

◆ external_relay_memory

bool external_relay_memory[4] = { false, false, false, false }

Sets external relay and open-collector (OC) outputs on PoKeys devices.

Configures external relay and OC outputs for Pulse Engine v2–enabled PoKeys devices. The function supports different output bit mappings based on the detected device type:

  • PoKeys57CNC
  • PoKeys57CNCpro4x25
  • PoKeysCNCaddon (default)

Procedure:

  1. Reads the current device output states via PK_PEv2_ExternalOutputsGet().
  2. Updates corresponding HAL pin values with retrieved states.
  3. Composes desired output states based on HAL pins:
    • PEv2_digout_ExternalRelay_out[] for relay outputs
    • PEv2_digout_ExternalOC_out[] for OC outputs

Device-specific bit mappings (based on PoKeys protocol specification v14.3.2025):
