Detailed Description

Canonical HAL interface helpers for incremental encoders.

This group provides export and registration functions for encoder devices following the LinuxCNC Canonical Device Interface (CDI).

Export canonical encoder HAL pins and parameters.

Export HAL pins and parameters for a canonical encoder channel.

Creates all HAL pins and parameters as defined by the Canonical Device Interface (CDI) for encoder devices. The pin names follow the format: <prefix>.encoder.<index>.<field>

Parameters
encPointer to the encoder struct to populate.
prefixCanonical name prefix, e.g. "enc.0" or "hw.1".
indexChannel number (0-based) within the device.
comp_idHAL component ID from hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html#sec:hal-cdi:encoder

Register periodic read function for a canonical encoder channel.

Registers a HAL function named <prefix>.encoder.<index>.read that is expected to:

  • Capture the hardware encoder count
  • Update the canonical count, position, and velocity pins

The registered function will be called cyclically from the HAL thread context and must match the signature: void read_func(void *inst, long period_nsec);

Parameters
prefixCanonical name prefix (e.g. "enc.0")
indexChannel number (0-based)
read_funcFunction pointer to the implementation
instContext pointer (typically a pointer to hal_encoder_t)
comp_idHAL component ID returned by hal_init()
Returns
0 on success, or a negative error code on failure.
Note
Floating-point math is enabled (uses_fp = 1).
