Detailed Description

Export and registration helpers for canonical digital input/output.

Export helpers for canonical digital input/output HAL interfaces.

This group contains functions to export standard HAL pins and parameters for digital input and output interfaces, as defined by the LinuxCNC HAL canonical device model.

Export canonical digin pins & parameter.

Export canonical digital input pins for a specific device channel.

This function creates the HAL pins required by the canonical digital input interface. The generated pin names follow the format: <prefix>.digin.<index>.in and <prefix>.digin.<index>.in-not

The <prefix> typically includes the device name and device number (e.g. "gpio.0"). The <index> corresponds to the input channel number within the device.

Parameters
diginPointer to the digital input struct instance.
prefixDevice-specific prefix, usually of the form "<device-name>.<device-num>".
indexChannel number (e.g. 0 for the first input of the device).
comp_idHAL component ID, as returned by hal_init().
Returns
0 on success, or a negative error code on failure.
Note
This function only creates HAL pins. It does not register any HAL function.
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html

Export canonical digout pins and parameters.

Export canonical digital output pin and parameter for a specific device channel.

This function creates the HAL output pin and the associated "invert" parameter as defined by the canonical digital output interface. The generated names follow this format: <prefix>.digout.<index>.out and <prefix>.digout.<index>.invert

The <prefix> usually includes both the device name and its instance number (e.g. "relay.1"). The <index> refers to the specific output channel.

Parameters
digoutPointer to the digital output struct instance.
prefixDevice prefix (e.g. "relay.0", "gpio.2", etc.).
indexOutput channel number for the device.
comp_idHAL component ID from hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
Note
No HAL function is registered. Functional logic must be registered separately.
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html

Register a HAL read function for a canonical digital input instance.

Registers a HAL function named <prefix>.digin.<index>.read, which is intended to read the physical input and update HAL pins in and in-not.

Parameters
prefixCanonical device prefix (e.g. "gpio.0")
indexChannel index (e.g. 0)
read_funcFunction to call each cycle, must match signature: void (*)(void*, long)
instPointer passed to the function (e.g. context or digin struct)
comp_idHAL component ID from hal_init()
Returns
0 on success, or a negative HAL/RTAPI error code.
Note
The function must be non-NULL. Floating-point is enabled by default.
