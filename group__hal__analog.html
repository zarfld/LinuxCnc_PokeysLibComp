Detailed Description

Register canonical digout write function.

Canonical interface helpers for analog input and output devices.

This module provides HAL-compatible data structures and functions for exporting canonical analog inputs (adcin) and outputs (adcout) as defined by the LinuxCNC Canonical Device Interface (CDI).

Analog Input (<tt>adcin</tt>)

  • Provides a scaled floating-point representation of analog hardware input.
  • Canonical pins:
    • value (HAL_OUT, float): scaled analog input value
  • Canonical parameters:
    • scale, offset, bit-weight, hw-offset (HAL_RW, float)
  • Canonical function:
    • read – updates value from hardware, applying scale and offset

Analog Output (<tt>adcout</tt>)

  • Represents a continuous analog output signal, e.g. for DAC or PWM hardware.
  • Canonical pins:
    • value (HAL_IN, float): value to write to hardware
    • enable (HAL_IN, bit): when false, output is forced to 0
  • Canonical parameters:
    • offset, scale, high-limit, low-limit, bit-weight, hw-offset (HAL_RW, float)
  • Canonical function:
    • write – reads value and parameters, writes scaled output to hardware

See also: https://linuxcnc.org/docs/html/hal/canonical-devices.html

Export HAL pins and parameters for a canonical analog input channel.

Export canonical adcin pins and parameters.

This function creates all HAL entities required for a single analog input (adcin) channel as defined in the Canonical Device Interface (CDI).

The following names will be created (formatted as <prefix>.adcin.<index>.<field>):

  • value (HAL_OUT, float): the scaled analog input value
  • scale (HAL_RW, float): multiplier applied to the raw hardware value
  • offset (HAL_RW, float): added after scaling
  • bit-weight (HAL_RW, float): granularity of input resolution (LSB size)
  • hw-offset (HAL_RW, float): value present at the input for 0V (calibration)
Parameters
adcinPointer to the analog input struct to be populated.
prefixCanonical name prefix, e.g. "adc.0" or "device.2".
indexChannel number (0-based) within the device.
comp_idHAL component ID as returned by hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html#sec:hal-cdi:analog-in

Export HAL pins and parameters for a canonical analog output channel.

Export canonical adcout pins and parameters.

This function creates the HAL pins and parameters needed to implement a single canonical analog output (adcout) as specified in the LinuxCNC Canonical Device Interface.

The following HAL names are generated, using the format <prefix>.adcout.<index>.<field>:

Pins

  • value (HAL_IN, float): input signal that defines the analog output level
  • enable (HAL_IN, bit): when false, output is forced to 0

Parameters

  • offset (HAL_RW, float): added to value before hardware write
  • scale (HAL_RW, float): multiplier applied before write
  • high-limit (HAL_RW, float): upper clamp value (optional)
  • low-limit (HAL_RW, float): lower clamp value (optional)
  • bit-weight (HAL_RW, float): LSB resolution of the DAC (optional)
  • hw-offset (HAL_RW, float): physical value produced when output is 0 (e.g., 4 mA)
Parameters
adcoutPointer to the analog output struct to populate.
prefixCanonical name prefix, e.g. "dac.0" or "analog.1".
indexOutput channel number (0-based).
comp_idHAL component ID as returned by hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html#sec:hal-cdi:analog-out

Register a periodic HAL read function for a canonical analog input.

This function registers a HAL function named <prefix>.adcin.<index>.read which should read a raw hardware analog value and update the scaled value pin.

The user-defined read_func is called cyclically by the HAL scheduler and is responsible for applying the scaling logic (scale, offset, bit-weight, hw-offset) and writing the result to adcin->value.

The function must follow the HAL function signature: void read_func(void *inst, long period_nsec);

Parameters
prefixCanonical device prefix, e.g. "adc.0".
indexChannel number (e.g. 0 for the first input).
read_funcUser-defined function to call every cycle.
instPointer passed to the function (e.g. hal_adcin_t* or user context).
comp_idHAL component ID from hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
Note
Floating-point math is enabled (uses_fp = 1).
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html#sec:hal-cdi:analog-in

Register a HAL write function for a canonical analog output channel.

Registers a periodic HAL function named <prefix>.adcout.<index>.write that should read the HAL pin value, apply the defined parameters (e.g. scale, offset, limit), and write the final result to hardware.

The user-defined function must have the signature: void write_func(void *inst, long period_nsec);

This function will be called automatically by the HAL scheduler in the context of the real-time thread (typically base-thread or servo-thread). The argument inst is a user-defined context, typically a pointer to the hal_adcout_t structure or a device-specific context.

Parameters
prefixCanonical device prefix, e.g. "dac.0".
indexOutput channel index within the device.
write_funcPointer to the user-defined function to register.
instPointer passed to the function (usually hal_adcout_t* or similar).
comp_idHAL component ID returned by hal_init().
Returns
0 on success, or a negative HAL/RTAPI error code on failure.
Note
Floating-point operations are enabled (uses_fp = 1).
See also
https://linuxcnc.org/docs/html/hal/canonical-devices.html#sec:hal-cdi:analog-out
