Detailed Description

Network transport helpers for PoKeys devices.

Implements discovery and packet transfer over UDP/TCP as described in the PoKeys protocol specification.

◆ PK_SearchNetworkDevices()

int32_t PK_SearchNetworkDevices ( sPoKeysNetworkDeviceSummary *  devices,
uint32_t  timeout,
uint32_t  serialNumberToFind 
)

Broadcasts a discovery request and collects responses from PoKeys devices.

A UDP packet is sent to every broadcast address returned by GetBroadcastAddresses(). Responses are gathered for up to timeout milliseconds and stored into the provided devices array (maximum 16 entries). If serialNumberToFind is non-zero the function stops early when a matching device is found.

Parameters
devicesArray of sPoKeysNetworkDeviceSummary structures for storing discovered devices.
timeoutDiscovery timeout in milliseconds.
serialNumberToFindOptional serial number to look for (0 = any).
Returns
Number of discovered devices.

◆ SendEthRequest()

int32_t SendEthRequest ( sPoKeysDevice *  device)

Sends a single request packet and waits for its 64 byte response.

The packet contained in device->request is transmitted using either UDP or TCP depending on the connection parameters. The function blocks using select()/recv() until a valid response is received or retries are exhausted.

Parameters
devicePointer to an initialized network sPoKeysDevice.
Returns
PK_OK on success or PK_ERR_TRANSFER on communication failure.

◆ SendEthRequestBig()

int32_t SendEthRequestBig ( sPoKeysDevice *  device)

Sends a multi-part request stored in device->multiPartData.

The data is split into eight packets (64 bytes each) and transmitted sequentially. After sending all parts the function waits for a single response packet. This is used when a request payload exceeds the standard 64 byte size.

Parameters
devicePointer to a connected sPoKeysDevice with a valid multi-part buffer.
Returns
PK_OK on success or PK_ERR_TRANSFER on timeout or communication errors.
