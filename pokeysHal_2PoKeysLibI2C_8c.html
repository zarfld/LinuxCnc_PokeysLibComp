Detailed Description

I\2C communication helpers (command 0xDB).

These functions wrap the PoKeys I\2C command set and transfer data between the host and a connected PoKeys device.

Obtain the results of an I\2C bus scan (command 0xDB/0x31).

After PK_I2CBusScanStart() completes, this helper reads the scan bitmap and stores device presence information into presentDevices.

Parameters
deviceHandle to an opened PoKeys device.
statusReceives the scan status (see ePK_I2C_STATUS).
presentDevicesArray that receives one byte per address indicating whether a device acknowledged.
iMaxDevicesMaximum number of addresses to report (up to 128).
Returns
PK_OK on success, PK_ERR_NOT_CONNECTED if device is NULL or PK_ERR_TRANSFER on communication failure.

Obtain the result of an I\2C read command (command 0xDB/0x21).

After a read has been started with PK_I2CReadStart() or PK_I2CWriteAndReadStart(), this function polls the device for completion and copies the returned bytes into buffer.

Parameters
deviceHandle to an opened PoKeys device.
statusReceives the operation status (see ePK_I2C_STATUS).
iReadBytesNumber of bytes actually read on success.
bufferDestination buffer for the received data.
iMaxBufferLengthSize of buffer in bytes.
Returns
PK_OK on success, PK_ERR_NOT_CONNECTED if device is NULL, PK_ERR_TRANSFER on communication failure or PK_ERR_GENERIC when the device reports more data than fits into 32 bytes.

Write data and queue a subsequent read (command 0xDB/0x10).

This combines a write and read phase. Up to 32 bytes are written from buffer and the device is instructed to read iDataLengthRead bytes afterwards. The read data is obtained later with PK_I2CReadStatusGet().

Parameters
deviceHandle to an opened PoKeys device.
addressI\2C slave address.
bufferData to write.
iDataLengthWriteNumber of bytes to write (0–32).
iDataLengthReadNumber of bytes expected to be read back.
Returns
PK_OK on command acceptance or PK_ERR_NOT_CONNECTED when device is NULL.
