43/* The update() function does all the work. If 'enable' is true, it
44 computes a new value of 'curr_pos', which moves toward 'pos_cmd'
45 while obeying the 'max_vel' and 'max_accel' limits. It also sets
46 'active' if movement is in progress, and clears it when motion
47 stops at the commanded position. The command or either of the
48 limits can be changed at any time. If 'enable' is false, it
49 ramps the velocity to zero, then clears 'active' and sets
50 'pos_cmd' to match 'curr_pos', to avoid motion the next time it
51 is enabled. 'period' is the period between calls, in seconds.
52*/
